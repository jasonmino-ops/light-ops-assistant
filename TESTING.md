# 轻经营助手 v1 — 测试版说明

## 启动

```bash
npm install
npm run dev
# 访问 http://localhost:3000
```

切换身份（修改后重启服务）：

```bash
# .env
DEV_ROLE=STAFF   # 首页 → /sale
DEV_ROLE=OWNER   # 首页 → /dashboard
```

---

## 已实现功能

| 页面 | 路径 | 说明 |
|------|------|------|
| 销售页 | `/sale` | 扫码/输条码 → 价格只读回填 → 填数量 → 提交 |
| 退款页 | `/refund` | 输原单号 → 查原单 → 选数量+填原因 → 提交 |
| 记录页 | `/records` | 日期范围 + 类型筛选，summary 跟随筛选联动 |
| 老板概览页 | `/dashboard` | 三维度汇总：全局 / 单门店 / 单销售员 |
| 首页 | `/` | 按 DEV_ROLE 自动跳转 |
| 底部导航 | 全局 | 固定底部 Tab Bar，当前页高亮 |

## 手工验证步骤

### 销售页（DEV_ROLE=STAFF）

1. 访问 `/sale`
2. 输入条码 `8888001`，按 Enter 或点"查询"
3. 确认：商品名/规格/售价只读回填，价格不可编辑
4. 数量改为 `2`，确认小计显示 `$5.00`
5. 点"提交销售"
6. 确认：顶部出现绿色结果卡（单号 + 金额 + 时间）
7. 点"清空"，确认表单重置

### 退款页（DEV_ROLE=STAFF）

1. 访问 `/refund`
2. 输入 `S-SEED-STORE-A-0001`，点"查找"
3. 确认展示：矿泉水 / 已售 3 / 已退 0 / 可退 3（红色）
4. 退款数量填 `1`，退款原因填"测试退款"
5. 点"确认退款"
6. 确认：顶部出现绿色结果卡（退款单号 + 负金额）
7. 再次查找 `S-SEED-STORE-A-0001`，确认可退数量变为 `2`
8. 边界验证：退款数量填 `99`，提交，确认返回"超出可退数量"错误

### 记录页（DEV_ROLE=STAFF）

1. 访问 `/records`，确认默认加载今日记录
2. 切"销售" → 确认 summary 的 refundCount=0，netAmount 只含销售额
3. 切"退款" → 确认 summary 的 saleCount=0，netAmount 只含退款额（负数）
4. 切"全部" → 确认两者均有，netAmount = 销售额 + 退款额
5. 修改日期范围到过去某天，确认列表和 summary 随之变化

### 老板概览页（DEV_ROLE=OWNER）

1. 设 `DEV_ROLE=OWNER`，重启，访问 `/dashboard`
2. 默认"全局汇总"，选今日日期，点"查询"
3. 确认：净收入 / 销售总额 / 退款总额 / 笔数 / 客单价 均有值
4. 切"单门店"，选"总店"，查询 → 确认只含总店数据
5. 切"单门店"，选"分店"，查询 → 确认只含分店数据
6. 切"单销售员"，选"小张（总店）"，查询 → 确认只含小张数据
7. 跨日期测试：dateFrom 设为较早日期，确认汇总包含历史数据
