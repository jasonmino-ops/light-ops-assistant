# 轻店助手 — 内测冻结基线 V1

> 本文件是当前可内测版本的唯一运行说明。功能已冻结，不再新增，仅做缺陷修复。

---

## 一、环境配置

复制并填写环境变量：

```bash
cp .env.example .env
# 编辑 .env，填入真实的 DATABASE_URL
```

`.env` 必填项：

| 变量 | 说明 | 示例 |
|------|------|------|
| `DATABASE_URL` | PostgreSQL 连接串（Neon / Supabase / 本地） | `postgresql://user:pass@host/db?sslmode=require` |
| `DEV_ROLE` | 本地角色切换，`STAFF` 或 `OWNER` | `STAFF` |

**Vercel 部署**：在 Project Settings → Environment Variables 添加 `DATABASE_URL`，无需 `DEV_ROLE`（生产环境后续接入真实登录后替换）。

## 二、首次启动

```bash
# 1. 安装依赖（自动执行 prisma generate）
npm install

# 2. 同步数据库结构（首次或迁移后执行一次）
npx prisma migrate deploy

# 3. 注入种子数据（幂等，可重复执行）
npx prisma db seed

# 4. 启动开发服务
npm run dev
# → http://localhost:3000
```

## 五、内测验证清单

### 5.1 销售流程（`DEV_ROLE=STAFF`）

1. 访问 `/sale`，输条码 `8888001` 回车
2. 确认商品名 / 规格 / 售价只读回填，价格不可编辑
3. 数量改为 `2`，确认小计 = `$5.00`
4. 点"提交销售"，确认顶部绿色结果卡出现（单号 + 金额）
5. 点"清空"，确认表单全部重置

### 5.2 退款流程（`DEV_ROLE=STAFF`）

1. 访问 `/refund`，输 `S-SEED-STORE-A-0001`，点"查找"
2. 确认展示：矿泉水 / 已售 3 / 可退 3（红色徽章）
3. 数量填 `1`，原因填"测试退款"，点"确认退款"
4. 确认绿色结果卡出现，金额为 `-$2.50`
5. 再次查找同单号，可退数量应变为 `2`
6. 边界：数量填 `99`，确认返回"超出可退数量"错误

### 5.3 记录页（`DEV_ROLE=STAFF`）

1. 访问 `/records`，默认今日，确认种子销售单出现
2. 切"退款"筛选，确认列表只含退款，汇总 saleCount = 0
3. 切"全部"，确认 netAmount = 销售额 + 退款额
4. 修改日期范围，确认列表和汇总随之联动

### 5.4 老板概览（`DEV_ROLE=OWNER`）

1. 访问 `/dashboard`，确认**无需点按钮**，今日全局数据自动加载
2. 确认同时出现：净收入 Hero / 销售额 / 退款额 / 销售单数 / 退款单数
3. 切"单门店"→ 选"总店"，确认数据自动刷新，Sub label 显示"总店"
4. 切"单店员"→ 选"小李（分店）"，确认只含小李数据
5. 确认"热销商品 Top 3"出现排名列表；无数据时显示"今日暂无销售记录"

### 5.5 权限隔离

1. `DEV_ROLE=STAFF` 直接访问 `/api/summary` → 返回 403
2. `DEV_ROLE=OWNER` 访问 `/api/records` → 返回全局记录（不被 storeId 硬限）

## 六、已知限制（V1 不修复）

| 项目 | 说明 |
|------|------|
| 无真实登录 | 当前用 `x-*` header 模拟身份，无密码校验 |
| 门店/员工为前端常量 | Dashboard 下拉写死在代码中，未接动态 API |
| 时区为 UTC | 跨时区门店日期边界可能偏差 |
| 高并发单号安全 | recordNo 生成存在竞争，生产需 DB SEQUENCE |
| Telegram 推送未实现 | 后续版本 |
