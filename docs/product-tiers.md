| **定位** | 初次体验数字化，零门槛上手 | 单店实体零售，日常稳定使用 | 多门店连锁，统一经营管理 |

## 扫码策略定义

### 统一底线规则（所有档次适用）

| 规则 | 说明 |
|------|------|
| 默认扫码方式 | 手机摄像头扫描商品条码（一维码） |
| 二维码扫描 | 暂不开通（所有档次） |
| 摄像头退出行为 | 未扫码成功前不得自动闪退，需等待用户主动操作 |
| 连续失败兜底 | 连续 5 次未识别成功后，提示用户手动输入条码 |

### 各档扫码策略

#### 轻试用版 `LITE`

| 维度 | 策略 |
|------|------|
| 主扫码方式 | 手机摄像头（默认且强制，无外设选项） |
| 扫码成功行为 | 结果停留 0.8～1.2 秒后继续，避免误触 |
| 兜底方式 | 手动输入条码 |
| 不支持 | HID 蓝牙扫码枪、二维码 |

#### 标准收银版 `STANDARD`

| 维度 | 策略 |
|------|------|
| 主扫码方式 | 外接 HID 蓝牙扫码枪（默认优先） |
| 输入框行为 | 进入销售页默认聚焦输入框，直接接收 HID 扫码枪输入 |
| HID 失败兜底 | 连续 5 次 HID 扫码未识别成功后，提示切换到手机摄像头扫码 |
| 摄像头扫码 | 仅扫商品条码，不扫二维码 |
| 最终兜底 | 手动输入条码 |

#### 门店标准化版 `MULTI_STORE`

| 维度 | 策略 |
|------|------|
| 扫码规则 | 完全继承标准收银版策略 |
| 打印/贴标能力 | 单独定义（v1+ 迭代），当前版本暂不包含 |

### 当前实现状态

> 以上为产品定义与规则约束，当前 v0.x 代码实现以摄像头扫码为主（`@zxing/browser`），HID 扫码枪通过键盘事件接收，连续失败检测与确认停留逻辑规划在后续迭代补齐。

## 暂不包含（各档共同）

以下功能当前版本（v0.x）均不包含，规划在 v1+ 迭代：

- 库存管理（进货/盘点/低库存预警）
- 小票打印（蓝牙热敏打印机接入）
- 高级 BI / 自定义报表导出
- 外部系统集成（ERP / 电商平台）
- 自动计费与订阅管理
- 顾客会员管理

---

## 档次存储与管理

- **数据库字段**：`Tenant.tier`（`String`，默认 `LITE`）
- **有效值**：`LITE` | `STANDARD` | `MULTI_STORE`
- **变更入口**：`/ops/[tenantId]` → 产品档次区块，运营管理员可一键切换
- **API**：`PATCH /api/ops/tenants/[tenantId]` `{ tier: string }`
- **当前版本**：仅展示与记录，无功能开关联动

## 商户档次分配建议

| 场景 | 建议档次 |
|------|----------|
| 新商户刚注册，尚未付费 | `LITE` |
| 已确认单店使用，完成商品录入 | `STANDARD` |
| 有 2 个以上门店或明确连锁需求 | `MULTI_STORE` |
